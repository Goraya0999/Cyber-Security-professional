# 🛡️ Bug Bounty - Day 1 Notes

---

## 🔍 What is Bug Bounty?

Bug bounty is a program where companies pay people to find security bugs in their systems.  
These bugs can be in websites, apps, or software.  
Ethical hackers (security researchers) report these bugs and get rewards.  
It helps companies stay secure and protect user data.  

---

## ❓ Why do we need Bug Bounty?

Bug bounty helps find real security problems before hackers do.  
Companies cannot find all bugs by themselves.  
It allows many experts to test systems worldwide.  
It improves security and protects users.  
It also builds trust between users and companies.  

---

## 📈 Demand in Future

Bug bounty demand is increasing day by day.  
More companies are going online and need security.  
Cyber attacks are also increasing.  
So, skilled bug hunters will be needed more in future.  
It is a good career option in cybersecurity.  

---

## 🤖 AI Affecting Bug Bounty

AI is changing bug bounty in many ways.  
AI tools can help find bugs faster.  
They can automate scanning and testing.  
But attackers can also use AI to find vulnerabilities.  
So, learning both manual and AI-based skills is important.

## 🔐 OWASP Top 10 (Introduction)

OWASP Top 10 is a list of most common web security risks.  
It helps developers and hackers understand major vulnerabilities.  

### Common Risks:
- Broken Access Control  
- Cryptographic Failures  
- Injection (SQL, Command, etc.)  
- Insecure Design  
- Security Misconfiguration  
- Vulnerable Components  
- Authentication Failures  
- Software Integrity Failures  
- Logging Failures  
- Server-Side Request Forgery (SSRF)  

---

## 🧪 OWASP Penetration Testing Checklist

- Information Gathering  
- Scanning and Enumeration  
- Vulnerability Analysis  
- Exploitation  
- Post Exploitation  
- Reporting  

Check for:
- SQL Injection  
- XSS (Cross-Site Scripting)  
- CSRF  
- Authentication issues  
- File upload vulnerabilities  
- Misconfigurations
